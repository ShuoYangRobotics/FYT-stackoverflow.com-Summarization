#0

The Agreement about Apple not accepting any coding language layer has been removed not too long ago. I guess we will have to wait a little to see complex language like Python and Ruby interpreter. Since LUA is a scripting language, it is easier to port it.


#1

What is your goal? If it is to write in a dynamic language instead of objective-C then you may want to investigate the Rhodes framework. It allows you to use Ruby in a Rails-like MVC framework and will create native iPhone and Android apps.


#2

If you just want to be able to run ruby and python scripts, codetogo lets you do that.

http://itunes.apple.com/us/app/codetogo/id382677229?mt=8

It gets around it by sending your code to a server, running it and then sending back the result.
