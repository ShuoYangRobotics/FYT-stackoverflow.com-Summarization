#0

Learn Python. It is easier to learn, has libraries to do virtually anything you might need, and can do essentially anything.

The only reason to use a more difficult language like C/C++ is if you need the performance or are writing code for an embedded system. They are not, however, what you should be learning initially.

C# is a fine language, but nothing beats Python for ease of use.

The scope of Python is quite broad, here are some examples:

  • Create a website (Django, etc.)
  • Create scripts to do tasks ranging from image manipulation to server maintenance
  • Create GUIs (Tkinter, etc.)
  • Create games (pygame)
  • Scientific computing (SciPy)

Python can interact directly with arbitrary C code, meaning anything which can be done in C, can be done in Python with a little work. Python is popular enough that an interface has been created for virtually everything already.

For a better look at what can be done with python out of the box, take a look at the standard library which comes with python: http://docs.python.org/library/

Don't get overwhelmed, most of us don't touch 90% of those, but they're there if you need them. There are also thousands upon thousands of projects which use or interact with Python in some way. Start with the tutorial. It's always helpful to have a goal when your learning something new, so make a website with CherryPy or a game with pygame.

In short, if it can be done with a computer, and doesn't require the speed of C/C++, it can be done with Python.

  • going from dynamic to strongly typed

  • having to do all of this at the same time and not being able to tell at first what was the actual C# language and what was each of these other aspects

Although I don't use Python, I've heard only great things about it and even in my ignorance have found no reason not to recommend it to others.

Learning C will expand your horizons in amazing ways and I highly recommend it. That said, I never use it for anything practical.

I have been a developer using all kinds of different languages since 1995 across a wide range of projects (hardware drivers to applications to games). This year I realized that unless I had a very compelling reason otherwise, all my future personal projects will be web applications (with no server side if it's avoidable). And I'm not talking about HTML pages with a bit of Javascript, I mean full fledged web applications. After developing noVNC (HTML5 VNC client) and seeing the existing and growing potential of web applications, there is just so little upside to developing fat client applications anymore (not unless I'm paid well to do otherwise).

I have deep and idiomatic knowledge of C, python, JavaScript, C++, perl, shell, etc and good experience with Fortran, Java, MIPS and x86 assembly, PHP, SQL, etc. And I'm currently learning Ruby and Clojure. Each of those languages has applications domains that they are very good for. But web application technologies (and JavaScript in particular) are quickly moving into most of those domains and the rate is only accelerating.

And before you dismiss web applications due to preconceived notions about what the web was (or is), I highly recommend this entertaining video of Paul Irish at jsconf.eu on the state of HTML5: http://blip.tv/file/4293357

So my recommendation would be learn all the technologies grouped under the Web Applications 1.0 banner (including HTML5).

#9

C# has a very nice IDE and library and is great for GUI applications. It is a better Java, and there is nothing at all wrong with that. I once had an interview at Microsoft in which the manager I would be working for asked me to write a simple application in C#. Never having used the language before (or Visual Studio, for that matter) I went from a standing start to a fully functional RSS reader that could auto-post to my blog in a single working day. (I landed the contract.)

Still, I love Python. Python (at least for me) has the advantage of producing code that works pretty much the way I intended the first time I run it. I think it's an excellent language to take your programming to the next level. It has a great standard library that has ready-to-use solutions to many higher-level programming situations, and just a ton of add-on modules that are ready to go. Also, there are some awfully sharp Python programmers here on SO to help you out when you get stuck. (I am sure there are also sharp C# programmers here, I just don't venture into their territory very often.)

Another nice thing about Python is that it supports multiple programming styles and makes it easy to transition among them. If you want to pretend you're writing a fancy shell script that is executed from top to bottom, Python will happily allow that. If you want to add a few functions to reduce the redundancy in your code, that is easily done. If you then decide that you want to wrap it up in a class so that you can easily use the functionality in other scripts, that too is straightforward. Python also has some nice functional-style elements, which will teach you new ways of thinking about solving problems. (I recently started reading an introduction to Haskell, a pure functional language, and I was already familiar with about half the concepts it brought up, thanks to Python.)

I also think the flexible type system is great for interactive experimenting and learning. It is very hard to experiment in C# because there's no interactive mode that lets you just enter commands and see what they do. Visual Studio's debugger comes close to that, though.

C and C++ are languages you will eventually want to know. But they require you to understand and keep track of a lot of the details that Python and C# handle for you, and a lot of the things you would take for granted in the Python or C# libraries, you would need to write from scratch in C++ (or C, if it was even practical to do in C). Learning C++ would be like being thrown into a lake to learn how to swim. You will end up a good programmer -- or you will quit programming entirely and take up some other vocation. I would start with a language that gives you water wings.
