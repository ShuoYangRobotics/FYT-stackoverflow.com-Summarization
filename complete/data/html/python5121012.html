#1

With the advent of better Python packaging tools (pip, virtualenv, etc.) I think you'd be happy on the Mac. PyObjC, py2app and appscript provide nice integration with Mac specific services, and of course you've got the standard Unix stuff underneath. Apple has bundled current versions of Python for years (2.6.1 is in OS X 10.6) and some of their server software, e.g. Calendar Server, is written in Python. There are some pretty serious Mac apps written in Python, such as Checkout and EagleFiler.

Package management is not as mature on the Mac as Linux (particularly if you're used to Debian/Ubuntu) though it's more of a developer than a user issue. There's no standard system package manager as most Linux distributions use, but homebrew and MacPorts are pretty well-liked. Personally I am happy using the system Python and being a version behind, but you can get a current binary distribution from python.org which happily coexists with the system version, or use a package manager to install Python.

If you use complex Python packages with lots of dependencies, such as NumPy, pygame, etc., and don't want to compile all the dependencies from source, you generally have a choice between using a binary distribution with an existing Python (e.g. python.org) or a using a package manager, though at least with MacPorts that means installing a separate version of Python.

And of course if you really need Linux for anything you can always run it in a VM or reboot into it.

This is a very subjective choice. Both Linux and Mac are fine platforms to develop on, be it Python or any other language with good POSIX support.

My choice is a Mac, with TextMate for my main editing environment. But my reasons have very little to do with Python; I like the It Just Works setup a Macbook gives me, and I can focus on my work instead of spending time tinkering with Linux, which I am very likely to do too much of.


#3

I do almost all of my python development on a Mac (or rather several Macs) using Textmate. I have run into some difficulties installing various numerical/scientific packages (mostly due to dependency issues rather than something about python itself) and can attest to the fact that when I've tried to build the same modules/packages in Linux it was generally easier. If you can deal with some of Apple's quirky library setups, then in general the end-user experience is much better on the Mac IMHO.

In order to be productive, I need my environment, which consists of many factors. I want Vim with the syntax highlighting I prefer. I want my operating system with the shortcuts I have configured. I want my desk and my chair. I want the keyboard that I'm used to type with.
